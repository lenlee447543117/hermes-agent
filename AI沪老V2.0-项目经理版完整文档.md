# 一、项目概述（V2.0升级版）

## 1.1 项目核心目标（迭代升级）

在V1.0「语音交互+后台自动化代操作」解决老人“不敢点、点不到、记不住”数字鸿沟痛点的基础上，完成**产品形态与智能能力的本质升级**。依托Hermes独立容器化智能体架构与Claude长文本大模型能力，打造**专属、有记忆、有情感、可进化**的适老数字伴侣。不仅实现微信通话、助老打车等基础场景自动化，更通过用户习惯画像、方言交互、主动关怀、自主策略优化，解决高龄老人情感孤独、操作记忆衰退、使用适配难的深层痛点，实现工具属性到陪伴属性的升级，进一步夯实商业化与社会价值。

## 1.2 产品定位（全新定义）

国内首款**单用户单智能体实例**的适老化AI数字伴侣App。基于Hermes容器化独立架构，为每位高龄老人分配专属带记忆的AI智能体实例，融合上海方言情感交互、长期用户习惯记忆、自主进化优化、被动指令执行+主动智能关怀双重模式，兼顾工具实用性与情感陪伴属性，适配75岁以上高龄老人的认知、视觉、语言、行为习惯，同时满足子女远程监护、远程配置、政府智慧养老集采需求。

## 1.3 核心升级关键词

Hermes独立实例、容器化隔离、Agentic Workflow智能体工作流、Claude 3.5、长期用户记忆、习惯画像、主动关怀、上海方言交互、上下文多轮对话、按需弹性启停、自主进化优化、开发自动化、环境标准化、部署隔离化、VLA视觉反馈闭环、端云协同轻推理、子女远程同步配置

## 1.4 项目范围（V2.1进阶迭代新增范围）

### 1.4.1 保留V1.0+V2.0核心能力

- 无障碍服务+VLM+AutoGLM基础自动化架构
- 专属Hermes智能体实例、用户习惯画像体系、双模方言交互系统
- 多轮上下文记忆、智能体自主进化工作流、云端弹性部署机制
- 开发者AI自动化运维体系

### 1.4.2 V2.1全新进阶新增能力

- **标准化AI开发环境护栏**：落地.agentrules项目开发规范，统一AI代码生成、交互逻辑、UI、安全底线，实现开发环境标准化
- **全流程开发自动化体系**：搭建Agent专属自动化脚本库，实现方言测试、UI合规校验、日志分析、策略迭代全自动化

## 1.5 项目核心进化逻辑

V1.0：**被动工具型**——用户下达明确指令，App机械执行操作，无记忆、无思考、无情感，仅解决“操作难”问题。

V2.0：**主动陪伴型**——专属AI智能体记住用户习惯、偏好、日常行为，理解用户隐性需求，支持闲聊陪伴，主动发起关怀提醒，可自主优化适配策略，解决“操作难+孤独感+适配差”多重痛点。

V2.1：**生产级可进化型**——基于Agentic Workflow打通开发、部署、运维、迭代全流程自动化，实现环境标准化、部署隔离化、迭代智能化，同时补齐隐私安全、视觉精准执行、远程配置能力，让专属智能体从“可用陪伴”升级为“稳定、安全、持续进化的专属数字伴侣”。

# 二、产品需求（PRD）完整版（V2.0+V2.1进阶）

## 2.1 目标用户画像升级

在原有用户画像基础上，补充情感需求与方言使用特征，适配V2.0陪伴属性，V2.1新增子女用户远程配置诉求：

维度

核心老年用户（75岁+）

付费子女用户（30-55岁）

新增特征

多数熟悉上海方言，普通话表达不流利；独居比例高，存在情感孤独；操作习惯固化，学习能力弱，需要长期适配；行为规律固定（固定作息、固定使用习惯）

除安全监护外，新增**情感陪伴诉求**，希望AI缓解老人独居孤独，通过行为报告掌握老人日常状态；新增**远程配置诉求**，希望远程简化老人设备配置，降低老人操作负担

新增痛点

标准普通话交互不适应；单一工具功能无温度；设备操作反复出错，越用越焦虑；无人沟通、日常状态无人关注；设备配置复杂，无法自主完成初始化设置

无法实时了解老人日常行为状态；无法远程感知老人情绪与独居状态；无法远程帮老人配置常用功能、联系人，老人使用门槛高

新增诉求

方言交流、生活化闲聊陪伴、个性化操作适配、主动提醒关怀、极简初始化配置

获取老人每日行为画像报告、查看AI陪伴记录、远程自定义老人关怀规则、远程配置常用联系人与应用白名单

## 2.2 核心功能全新升级需求（含V2.1进阶）

### 2.2.1 核心能力升级：从指令执行到意图预测+主动关怀

V2.0彻底颠覆传统AI工具的被动交互模式，基于Hermes专属智能体的长期记忆能力，实现**被动执行+主动预判**双模式运行。V2.1进一步优化执行逻辑，通过VLA视觉闭环解决传统自动化脚本盲操作、适配性差的问题，实现精准可视化执行。智能体持续学习老人的作息习惯、操作偏好、语言习惯、行为规律，形成专属用户习惯画像。

**核心场景示例：**

- 老人日常固定早8点查看天气，若连续2天未操作，Hermes智能体主动用上海方言询问：“阿叔，今朝哪能没看天气啦？身体还好伐？”
- 老人常用微信视频联系女儿、语音联系儿子，智能体自动记忆偏好，老人模糊说“打个电话给囡恩”，自动匹配视频通话
- 老人多次调整播报音量，智能体记忆常用音量档位，后续默认适配用户偏好

### 2.2.2 双模交互体系（方言+双语、指令+闲聊）

#### 1、语言能力：上海方言+普通话双模适配

- **方言识别**：精准识别上海本地口语、俚语、模糊表达，支持带口音普通话、纯上海话指令输入
- **方言播报**：TTS支持上海方言语音回复，语气亲切生活化，贴合老年用户语言习惯
- **自动适配**：智能体根据用户历史对话，自动默认用户常用语言，支持手动切换普通话/上海话

#### 2、交互模式：严格指令模式 + 闲聊陪伴模式

- **严格指令模式**：识别用户操作类指令（打电话、打车、打开医保码），优先执行手机自动化操作，响应高效无冗余对话
- **闲聊陪伴模式**：识别情感类、闲聊类语句（聊家常、问旧事、吐槽生活），启动情感对话，不执行设备操作，提供情绪陪伴
- **模式智能切换**：智能体自动语义判断，无需用户手动切换，无缝适配用户需求

### 2.2.3 多轮上下文记忆交互

基于Redis缓存短期对话上下文，结合Hermes数据库长期记忆，实现连贯多轮对话，解决传统AI单次对话无记忆的痛点。V2.1优化记忆策略，由Claude定期总结近50条对话生成摘要，精简存储、提升上下文匹配效率。

**场景示例：**

- 用户：“声音有点小”→AI调高播报音量并方言回复
- 用户：“再大点声”→AI识别上下文，继续调高当前音量，无需用户重复说明需求
- 用户：“帮我打视频给儿子”→执行操作→用户：“换成语音吧”→AI基于上下文修改操作指令

### 2.2.4 专属用户习惯画像系统

Hermes智能体每日凌晨自动执行画像分析任务，基于当日用户所有操作记录、对话记录、使用行为，生成**老人专属每日状态报告**，包含作息状态、设备使用情况、操作难点、情绪倾向、异常行为预警。V2.1新增习惯权重体系，量化用户不同时段、不同场景的操作偏好，提升主动预判精准度。

**画像核心维度：**

- 作息习惯：常用使用时段、固定操作行为、作息是否规律
- 操作偏好：常用功能、常用联系人、音量/字体偏好、操作习惯、场景化习惯权重
- 交互偏好：方言/普通话偏好、闲聊频率、对话风格偏好
- 行为异常：长期未使用高频功能、操作频繁失败、长时间无设备交互

报告可同步展示在子女端，支持子女查看、设置关怀阈值，触发异常主动提醒。V2.1新增子女远程配置同步能力，支持自定义老人使用配置。

### 2.2.5 Hermes智能体自主进化能力

基于Agentic Workflow工作流，智能体自动复盘用户操作日志，迭代个性化适配策略，实现**越用越适配、越用越智能**。V2.1依托标准化开发护栏与自动化脚本，实现策略迭代全自动化，无需人工干预。

**进化场景：**

- 若老人连续3次手动重试“打开医保码”操作，智能体自动优化该场景的自动化识别策略，降低操作失败率
- 若老人多次使用方言模糊指令，智能体自动更新本地语义库，适配用户专属口语习惯
- 自动更新`.agentrules`策略文件，沉淀用户专属辅助规则，无需人工迭代代码

### 2.2.6 V2.1进阶核心新增功能

#### 1、前端VLA视觉反馈闭环能力

重构传统自动化执行逻辑，摒弃固定坐标脚本模式，实现「AI意图解析→屏幕视觉脱敏采集→VLM坐标识别→精准执行→结果反馈重试」的全闭环自动化能力，彻底解决机型适配、界面更新导致的操作失效问题，大幅提升自动化成功率。

#### 2、端侧实时隐私保护墙

在MediaProjection截屏链路中嵌入OpenCV实时脱敏滤镜，自动识别屏幕内人脸、数字账号、手机号、身份证号等隐私区域，上传云端前完成涂黑脱敏，实现隐私数据端侧清零，从源头保障用户数据安全，符合养老数据合规要求。

#### 3、子女端SyncConfig远程同步配置

Hermes实例新增统一配置同步接口，支持子女Web端/小程序远程配置老人设备参数，核心可配置内容包含：常用通话联系人、App使用白名单、主动关怀提醒频次、方言/普通话默认模式、设备音量字体默认参数，配置实时同步至老人专属智能体与端侧设备，大幅降低老人初始配置门槛。

AI沪老V2.1采用**Android感知外壳 + Termux/Hermes边缘中枢 + 云端推理**三层协同架构，实现"感知-决策-执行"完整闭环：

```

│                    Android 手机                      │
│  ┌──────────────────────────────────────────────┐  │
│  │         感知外壳 (Android Native App)         │  │
│  │  • 暖阳橙 UI / 机器人动画                     │  │
```

## 3.2 四层解耦架构详解

### 3.2.1 感知层（Perception Layer）

**职责：** 采集用户语音意图和屏幕视觉信息

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 离线唤醒 | Vosk + 300MB轻量模型 | "小沪小沪"唤醒词，零延迟本地识别 |

**数据流：** 用户语音 → Vosk唤醒检测 → 智谱ASR文字 → 意图解析

### 3.2.2 认知层（Cognition Layer - The Brain）

**职责：** 将自然语言意图转化为结构化执行指令

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 本地意图解析 | 规则引擎 + 关键词匹配 | 离线可用，<100ms响应 |

| 视觉理解 | VLM (Qwen-VL-Max) | 屏幕元素识别，坐标定位 |
| Prompt缓存 | LRU Cache + 预热 | 常用场景TTFT≤500ms |

**三级意图解析（与开发团队版一致）：**
```
Level 1: 本地动作缓存命中 → 直接回放动作链（<500ms）
Level 2: 离线规则匹配 → 关键词+正则匹配意图（<100ms）
Level 3: 云端AutoGLM推理 → 复杂语义理解+模糊指令解析（<2s）
```

**数据流：** 文字意图 → Level 1缓存命中(直接执行) → Level 2规则匹配(命中则执行) → Level 3云端推理 → 结构化JSON指令

### 3.2.3 执行层（Execution Layer - The Hands）

**职责：** 将结构化指令映射为物理操作

| 组件 | 技术方案 | 说明 |
|------|----------|------|

| uiautomator2 | 无线ADB连接 | Hermes端自动化引擎，更稳定 |
| 执行队列 | 单线程串行队列 | 系统中断优先，三步错误恢复 |

**三策略降级执行（核心创新）：**
```
策略1: AccessibilityService节点查找 → d(text="搜索").click()
策略2: VLM视觉识别坐标 → 截图→VLM→归一化坐标→手势点击
策略3: 固定坐标兜底 → 基于屏幕比例的归一化坐标
```

**职责：** 权限保活、冲突处理、安全监控

| 云端代理 | AutoGLM/VLM | 复杂语义推理、视觉定位 |

## 5.2 接口契约（对齐解决方案.md）

### 5.2.1 App → Hermes（Termux本地通信）

| `/api/v1/cancel` | POST | 空 | `{"status": "cancelled"}` | 中断当前任务 |

### 5.2.2 Hermes → 云端（按需调用）

## 5.3 执行策略决策树

```
收到任务 "帮我在微信里找到囡囡并视频通话"
    │
    ├─ 1. 检查本地动作缓存 → 命中则直接回放（<500ms）
    │
    ├─ 2. 解析意图 → "打开微信 → 搜索囡囡 → 点击视频通话"
    │
    ├─ 3. 逐步执行：
    │   ├─ "打开微信" → launch_app("com.tencent.mm")
    │   ├─ "搜索囡囡" →
    │   │   ├─ 尝试 d(text="搜索").click()  ← 文本匹配
    │   │   ├─ 失败 → 截图上传 VLM → 返回搜索框坐标 → 点击
    │   │   └─ 输入 "囡囡"
    │   └─ "点击视频通话" →
    │       ├─ 尝试 d(description="视频通话").click()  ← content-desc
    │       ├─ 失败 → 截图上传 VLM → 返回图标坐标 → 点击
    │       └─ 仍失败 → 本地模板匹配 → 点击
    │
    └─ 4. 验证结果 → 成功则缓存动作链 → 反馈给老人
```

- Vosk模型、VLM坐标库、本地意图模板通过云端下发
- App下载后存入Termux共享目录，Termux检测更新并重载

# 七、适老化细节实现要点

1. **方言支持**：录制常用沪语TTS短语（"正在找囡囡""稍等一歇"），通过本地音频播放，避免合成不自然
2. **大字体与高对比度**：App使用24sp以上字号，按钮最小触摸区域48dp
3. **动作回放优先**：第二次相同指令直接执行缓存，延迟<500ms，并语音提示"老快额，马上好"
4. **误操作保护**：执行期间屏蔽系统导航栏（需辅助功能），仅保留悬浮停止按钮
5. **暖阳橙视觉主题**：主色#FF9500、背景色#FFF8F0，符合适老化色彩规范
6. **多通道反馈**：语音+震动+视觉三重反馈，确保老人感知到操作状态
